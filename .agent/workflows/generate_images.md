# Generate Images Workflow

This workflow documents the end‑to‑end process for creating premium branding assets for the AIGestion website.

## Steps

1. **Install dependencies**
   ```bash
   pnpm add -D @google-cloud/aiplatform
   ```
2. **Configure credentials**
   - Ensure the service account _nanobanana_ JSON key is available locally.
   - Set `GOOGLE_APPLICATION_CREDENTIALS` env var to the path of the key.
   - Set `GCP_PROJECT` (or `GOOGLE_CLOUD_PROJECT`) to your GCP project ID.
3. **Run the generation script**
   ```bash
   npm run generate:images
   # or
   node scripts/generate-images.js
   ```
   This will create `aigestion_office.png`, `android_hero.png`, and `daniela.png` in `frontend/apps/website/public/images`.
4. **Commit the assets**
   ```bash
   git add frontend/apps/website/public/images/*.png
   git commit -m "feat: add premium branding assets"
   ```
5. **CI integration** (optional)
   - Add a step in your CI pipeline to run `npm run generate:images` and verify the files exist.
   - Fail the build if generation fails.

## Best Practices

- Keep the prompts in `scripts/generate-images.js` up‑to‑date with the brand bible.
- Regenerate assets whenever the logo or brand colors change.
- Store the generated assets in version control to ensure reproducibility.
